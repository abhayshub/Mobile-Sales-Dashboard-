# 📱 Mobile Store Analytics Dashboard

## Recommended Structure and Order

### 1. Project Title / Headline

# 📊 Mobile Store Analytics Dashboard: Sales & Customer Insights

An interactive Power BI dashboard designed to analyze mobile phone sales performance, customer behavior, payment methods, and brand-wise trends across different regions and months.

This dashboard helps businesses monitor sales KPIs, identify top-performing products, track customer ratings, and optimize business decisions using data-driven insights.

---

## 2. Short Description / Purpose

The Mobile Store Analytics Dashboard is a dynamic Power BI report created to provide a complete overview of mobile store performance. It enables users to analyze total sales, transaction volume, product demand, customer ratings, payment preferences, and monthly trends.

The dashboard is useful for mobile retailers, business owners, sales analysts, and decision-makers looking to improve revenue and understand customer purchasing behavior.

---

## 3. Tech Stack

The dashboard was built using the following tools and technologies:

• 📊 **Power BI Desktop** – Main data visualization platform used for creating interactive reports.<br>

• 📂 **Power Query** – Used for data cleaning, transformation, and preprocessing.<br>

• 🧠 **DAX (Data Analysis Expressions)** – Used to create calculated measures, KPIs, and dynamic aggregations.<br>

• 🔗 **Data Modeling** – Relationships created between sales, customers, products, and transactions tables.<br>

• 📈 **Interactive Visualizations** – Maps, KPIs, charts, slicers, and tables for business analysis.<br>

• 📁 **File Format** – `.pbix` for development and `.png` for dashboard previews.<br>

---

## 4. Data Source

**Source:** Mobile sales transactional dataset

Dataset contains information including:

- Mobile brands and models
- Sales amount
- Quantity sold
- Customer ratings
- Payment methods
- Transactions
- Monthly sales records
- Geographic location of sales
- Brand-wise performance metrics

The data is structured to allow filtering by month and mobile models.

# 5. Features / Highlights

## • Business Problem

Mobile retailers generate thousands of transactions monthly, making it difficult to identify:

- Which mobile brands perform best?
- Which months generate maximum revenue?
- What payment methods are preferred?
- Which products have highest sales volume?
- How customer ratings affect performance?
- Which regions contribute most to sales?

Answering these questions manually is time-consuming.

---

## • Goal of Dashboard

The dashboard aims to:

✔ Monitor overall sales performance

✔ Track monthly sales trends

✔ Analyze customer purchasing patterns

✔ Identify top-performing mobile brands

✔ Understand preferred payment methods

✔ Compare transaction volume and quantity sold

✔ Support data-driven business decisions
